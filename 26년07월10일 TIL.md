1. Vector 컨테이너

    Vector의 인상은 배열에 가까웠으며, 이는 실제로 Vector가 내부적으로 연속된 메모리를 사용하는 컨테이너이므로 배열과 비슷한 특징을 가진다. 
    또한 Vector는 뒤에서 삽입·삭제를 하는 것이 가장 효율적이므로 Stack처럼 사용하는 것이 가장 효율적이다.

    Stack: Stack은 가장 마지막에 들어간 데이터를 가장 먼저 꺼내는 자료구조(LIFO)이다. 하노이의 탑에서 원반도 항상 가장 위의 원반만 움직일 수 있으므로 각 기둥을 하나의 Stack처럼 생각할 수 있다. 원반을 옮기는 과정은 한 기둥에서 Pop하여 다른 기둥으로 Push하는 과정과 같다.


2. Vector 컨테이너에서 자주 사용하는 메서드
    
    push_back: 위에서 설명한 Stack의 Push의 기능을 담당하는 메서드로 Vector의 가장 마지막 위치에 값을 넣는다.

    pop_back: push_back과 반대로 이쪽은 Stack의 pop기능을 담당하는 메서드이다.
    
    Vector의 마지막 위치에서 값을 제거하는 메서드이다.

    size: 현재 Vector의 크기를 나타내는 값이다. 이 size는 for문을 이용한 루프 탐색에 사용할 수 있다.

    erase: Vector의 특정 위치에서 값을 지울 수 있는 메서드로, size와 후술할 반복자를 사용하여
    Vector 내부의 값을 서칭하고 제거하는데 사용할 수 있다.


3. Map 컨테이너

    key와 값을 가지는 컨테이너로 첫 인상은 노드를 흩 뿌려둔 자료구조의 Tree와 같았다.
    Tree의 데이터에서 특징을 추출한 것이 key이며, 이에 해당하는 데이터를 소유한 느낌이었다.
    하지만 이는 다른 내용으로 key는 데이터를 찾기 위한 기준이며, 이에 대응되는 value를 저장하는 형태였다.
    
    이후 추가로 조사한 결과로 Map은 내부적으로는 Tree(대표적으로 Red-Black Tree)를 이용하여 key를 정렬된 상태로 관리한다. 따라서 key는 비교 연산(<)이 가능해야 하며, 기본적으로 오름차순으로 저장된다.

    Tree: 부모와 자식 관계를 이용하여 데이터를 계층적으로 저장하는 자료구조로, 도서관이나 서점의 분류 시스템을 생각하면 편리하다. 도서관과 서점은 각 서적을 분류에 따라 제각각 나누고 여기에서 다시 이름별로 분류하는 경우가 많다. 

4. Map 컨테이너에서 자주 사용하는 메서드

    insert: Map에 값을 넣는 메서드로 사용하면 key의 정렬 순서를 유지하도록 적절한 위치에 삽입된다. insert = 끼워넣다를 생각하면 Vector의 push_back이나 다른 자료 입력과의 차이를 확실하게 기억할 수 있다.

    find: Map에서 값을 찾는 메서드로 알맞은 key를 이용하여 해당 요소를 찾을 수 있다.
    해당 key가 존재하지 않는다면, map.end()를 반환하며 이는 후술할 이터레이터에서 더욱 상세히 설명한다.

    size: Vector와 동일하게 크기를 반환하는 메서드이며, Vector와 다르게 Map은 Map[]에서 []안에
    key가 들어감으로, Vector와 동일하게 사용할 수 없다.

    erase: Vector처럼 특정 위치의 값과 키를 지우는 메서드로 key를 활용하여 특정 값을 정확하게 지울 수 있다.

    clear: Map의 값을 전부 지우는 메서드이다.

5. 알고리즘

    sort: 데이터를 정렬하는 알고리즘이며, 기본 자료형의 경우 기본은 오름차순으로 정렬되며
    sort(arr, arr + size, compare) 이 처럼 compare를 함수로 작성할 경우 compare 함수의 기준에 따라 정렬된다. 

    여기서 나는 sort(arr, arr + size, compare)에 compare를 () 표현 및 매개변수가 없이 표기 되는 것에 의문을 가졌는데, 이는 해당  sort 함수 내부적으로 compare 함수를 호출하기 때문이며, compare만 쓸 경우 함수가 구현되어 있는 주소로 연결된다는 것을 알았다.

    find: 컨테이너 내부에서 값을 찾는 알고리즘으로 Map에서 사용했던 메소드와 유사한 함수로 리턴 값이 같지만, 전혀 다른 함수 이다. 이는 주의가 필요하다. 이 함수는 본격적으로 반복자 활용에 용이한 알고리즘으로 (first, last, data)의 형태를 가지며 여기서 last는 마지막 값 +1을 필요로 한다.
    쉽게 말해 배열의 C의 배열에 적용한 sizeof나 위에서 사용한 메소드 size처럼 인덱스의 마지막 값+1이 정확한 표현이 되겠다. 여기서 언급하였던 반복자와 위에서 언급했던 iterator는 바로 다음 반복자에서 설명한다.
