1. 생성자(Constructor):
객체가 생성될 때 자동 호출된다. 

    예시: Archer(string name, int hp = 50, int mp = 50, int power = 50, int defence = 50):Player(name, hp, mp, power, defence) { this->job = "Archer"; }

    특징: 

    첫 번째:생성자를 사용할 때에는 기본 값을 지정하여 할당이 가능하다.

    두 번째:멤버 초기화 리스트는 생성자 본문보다 먼저 실행되어 멤버 변수와 부모 클래스 생성자를 초기화하는 문법이다.

2. 소멸자(Destructor)

    객체가 삭제될 때 자동 호출된다.

    예시: ~Player()
    
    소멸자의 주요 역할은 외부자원을 정리하는 것이며,
    메모리 누수 및 자원 누수를 방지하기 위해 사용한다.

    외부 자원이란?
    new, C언어의 malloc 등 처럼 Heap 메모리에 올라가는 자료들, 파일 핸들, 네트워크 소켓, DB 연결, 운영체제 리소스등 외부 자원을 정리하지 않으면 메모리 누수나 자원 누수가 발생할 수 있으며, 이는 성능 저하, 프로그램 오류, 시스템 자원 고갈로 이어질 수 있다.

3. 동적 할당 new와 해당 사항을 정리하는 delete

    new는 C++에서 활용하는 동적할당으로 사용하면 Heap 메모리에 클래스나 기본 타입의 사이즈에 맞추어
    메모리를 확보하며 대상이 클래스일 경우 생성자가 호출한다.

    반대로 delete는 확보한 Heap 메모리를 비우는데 사용하며, 클래스의 경우 소멸자가 호출된다.

    마지막으로 단일 객체는 new와 delete를 사용하고, 배열은 new[]와 delete[]를 짝으로 사용해야 한다.

    이 때 기본 틀이 되는 클래스를 기본 클래스 또는 부모 클래스라고 부르며
    상속을 받는 클래스를 파생 클래스 또는 자식 클래스라고 부른다.

5. 다형성과 가상 함수 및 순수 가상 함수

    파생 클래스가 여러 개 일 때, 같은 이름에서 비슷한 기능을 하거나 혹은 같은 기능이지만 다른 출력을 요구하는 함수를 만들어야 하는 상황이 존재한다.
    
    이럴 때 사용하는 것이 가상 함수이다, 가상 함수는 기본 클래스에서 선언하고 파생 클래스에 따라서 내용이 달라질 수 있는 함수로 다음 코드를 보면 이해하기 편하다.

    class Phone {
    public:
        virtual void displayBrand()  = 0; // 순수 가상 함수
        virtual void showFeature()  = 0; // 순수 가상 함수
        virtual ~Phone() {}
    };

    // 2. Samsung 클래스 작성 (Phone 클래스를 상속받음)
    class Samsung : public Phone {
    public:
        void displayBrand()   {
            cout << "Samsung" << endl;
        }
        void showFeature()  {
            cout << "Galaxy S 시리즈" << endl;
        }
    };

    // 3. Apple 클래스 작성 (Phone 클래스를 상속받음)
    class Apple : public Phone {
    public:
        void displayBrand()  {
            cout << "Apple" << endl;
        }
        void showFeature()  {
            cout << "iPhone Pro 시리즈" << endl;
        }
    };

    // 4. Xiaomi 클래스 작성 (Phone 클래스를 상속받음)
    class Xiaomi : public Phone {
    public:
        void displayBrand() {
            cout << "Xiaomi" << endl;
        }
        void showFeature() {
            cout << "Redmi Note 시리즈" << endl;
        }
    };

    int main() {
        Phone* phones[3];

        phones[0] = new Samsung();
        phones[1] = new Apple();
        phones[2] = new Xiaomi();

        for (int i = 0; i < 3; ++i) {
            phones[i]->displayBrand();
            phones[i]->showFeature();
        }

        for (int i = 0; i < 3; ++i) {
            delete phones[i];
        }

        return 0;
    }

    위의 코드를 보면 알 수 있는 것 처럼, 가상 함수를 사용하면 
    장점은 부모 타입의 포인터 배열에 서로 다른 자식 객체를 저장할 수 있고, 가상 함수 덕분에 실제 객체의 함수가 호출되어 코드의 재사용적인 측면에서 유리하다.

    또한 위에 사용하였던 예제에서 가상 함수가 상당히 특이하게 생겼는데, 이를 순수 가상 함수라고 한다.

    이 순수 가상 함수는 부모 클래스에서 함수의 형태(인터페이스)만 정의하고 구현은 자식 클래스에게 강제하는 기능이다. 순수 가상 함수가 하나라도 존재하면 해당 클래스는 추상 클래스가 되어 직접 인스턴스화할 수 없으며, 자식 클래스는 반드시 해당 함수를 구현해야 한다. 라고 정의되어 있다.

    그리고 이 말을 다시 한번 풀어보자면 부모 클래스는 공통 인터페이스를 제공하고, 자식 클래스는 각자의 방식으로 기능을 구현하므로 기능별 책임이 명확해진다. 이는 에러가 발생했을 때 기능에 따라 에러 발생 시 수정해야하는 부분에 대해 명확해 지며 이는 유지보수성에 큰 이점이다. 또한 이러한 유지 보수성에 큰 이점을 지니게 되면 코드 자체의 재사용적인 부분에서 큰 이점을 지니게 된다.

    따라서 여러 자식 클래스가 반드시 같은 기능 이름을 가져야 하고, 부모 클래스 자체는 객체로 만들 필요가 없을 때 사용하면 좋다.

6. 스마트 포인터

    변수나 함수 같은 Stack 메모리에 들어가는 내용은 프로그램이 알아서 관리해주어 편의성을 가지고 있었다. 

    하지만 malloc같은 Heap 메모리에 들어가는 동적할당을 직접적으로 delete(free)를 사용해줄 필요가 존재하였는데, 이는 실질적으로 코드를 작성하다 보면 까먹는 경우가 존재하여 메모리 누수의 원인이 되고는 하였다.

    따라서 스마트 포인터는 동적 메모리를 보다 안전하게 관리하고 메모리 누수를 줄이기 위해 등장하였다.

    스마트 포인터의 종류는 3가지가 있으며 기능은 아래와 같다.

    첫 번째로 unique_ptr, 두 번째로 shared_ptr, 마지막으로 weak_ptr이 있다.

    unique_ptr은 하나의 포인터만 객체를 소유, 범위를 벗어나면 자동 해제하는 포인터 이다.

    shared_ptr는 여러 포인터가 객체를 공동 소유, 참조 카운트가 0이 되면 자동 해제되며,
    내부적으로 참조 카운트를 관리하는 Control Block을 사용한다.
    
    weak_ptr은 객체를 소유하지 않고 관찰만 하며, 사용할 때는 대상 객체가 아직 살아 있는지 확인한 뒤 접근해야 한다.

    여기서 특이한 포인터는 weak_ptr인데 이 포인터는 shared_ptr를 많이 사용하다 보면 순환하게 되는 상황이 발생 할 수 있으며, 이를 해결하기 위해 사용한다.
    또한 weak_ptr은 객체를 소유하지 않고 관찰만 하며, shared_ptr의 참조 카운트에는 영향을 주지 않는다.
    단 사용할 때는 대상 객체가 아직 살아 있는지 확인한 뒤 접근해야 한다. 

7. 얕은 복사와 깊은 복사

    얕은 복사: 포인터에서 해당 주소만을 복사하는 방식(얕은 복사는 한쪽에서 메모리를 해제하면 다른 쪽 포인터가 댕글링 포인터가 될 수 있다.)
    깊은 복사: 새롭게 동적할당하여 Heap에 공간을 마련하고 값을 복사하여 저장하는 방식(깊은 복사는 클래스 내부에 포인터가 존재할 때 특히 중요하다.)

8. 언리얼 엔진의 GC(Garbage Collector)

    이 GC는 상당히 익숙했던 개념으로 파이썬을 습득할 때, 배운적이 있는 기능이다.
    
    이 기능은 스마트 포인터와 같은 이유에서 발전한 기능이며, 스마트 포인터를 활용 하지 아니하고도
    GC가 알아서 미 사용 중인 Heap의 기능을 정리해주는 기능이다.

    마지막으로 언리얼 엔진의 GC는 UObject 기반 객체 중 더 이상 참조되지 않는 객체를 탐지해 정리하는 시스템이다.

    다만 모든 Heap 메모리를 관리하는 것은 아니며, 일반 C++ 객체나 new로 직접 할당한 메모리는 GC가 자동으로 정리하지 않는다.

    오늘의 깨달음
    - Stack은 컴파일러가 관리하지만 Heap은 개발자가 관리한다(리마인드).
    - Smart Pointer는 Heap을 Stack처럼 안전하게 관리하기 위해 등장했다.
    - 순수 가상 함수는 기능 구현을 강제하여 인터페이스를 통일한다.
